# Listing Requirements Checklist

These criteria apply to the marketplace listing itself — the public-facing information customers see.

## Marketplace Information

- [ ] App name is clear, descriptive, and does not infringe trademarks
- [ ] Short description accurately summarizes what the app does
- [ ] Long description provides detail on features, use cases, and value
- [ ] Category and subcategory are appropriate
- [ ] Screenshots are current and consistent with actual app UI

## URLs and Links

- [ ] All documentation URLs are live and functional (not 404, not paywalled)
- [ ] Support URL leads to an active support channel
- [ ] Privacy Policy URL is accessible
- [ ] Terms of Service URL is accessible
- [ ] Setup/installation documentation URL works

## Setup Documentation

- [ ] Documentation is specific to the HubSpot integration (not generic product docs)
- [ ] Includes step-by-step installation instructions
- [ ] Includes configuration guidance
- [ ] Includes screenshots where helpful
- [ ] Describes what the app does within HubSpot specifically

## Pricing

- [ ] Pricing information on listing matches pricing on your website
- [ ] Free trial or freemium model clearly described (if applicable)
- [ ] No hidden costs or undisclosed pricing tiers

## Data Transparency

- [ ] "Shared data" table accurately describes data flow
- [ ] All data sent to/from HubSpot is disclosed
- [ ] Data usage purpose is clear to customers

## Support

- [ ] At least one support contact method is provided
- [ ] Support channel is actively monitored
- [ ] Response time expectations are reasonable

## Legal

- [ ] Compliant with HubSpot Acceptable Use Policy
- [ ] Branding and trademark guidelines followed
- [ ] No misleading claims about HubSpot partnership or endorsement
